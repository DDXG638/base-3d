# 项目背景

## 背景

我是一名有多年经验的 前端开发工程师，想系统地学习 web 3D 方向相关的技术知识，然后使用所学知识完成一个稍微复杂的 3D 项目，最终的目标是成为一名 3D方向的前端开发工程是。

## 职位描述

为了更清晰需要学的的 3D 相关的知识和技能，我从招聘网站上收集了几个 3D前端开发岗位 的招聘职位描述。希望可以根据这些职位描述中的技能有侧重地学习。

### xTool-3D前端工程师

岗位职责
1、负责 Web 3D 编辑器前端设计与开发
基于 React + TypeScript + Three.js 构建核心编辑器体验，覆盖场景搭建、对象选择、参数面板、视角操作、状态反馈、结果预览等关键链路。
2、搭建高质量的交互系统与视觉语言
把复杂的 3D 结构编辑过程转化为清晰、克制、具有美感的操作体验。你需要关注布局、层级、动效、反馈、空间感与信息密度，让工具既专业又易用。
3、与结构引擎协作完成参数化编辑体验
理解 panel、connection、joint 等底层结构概念，将引擎提供的数据和能力映射为前端可操作、可解释、可调试的交互界面，帮助用户完成参数修改、结构联动和结果验证。
4、参与 3D 数据展示与编辑链路设计
和产品、引擎、AI 一起定义从“创建结构”到“修改参数”再到“导出结果”的完整体验，推动 3D 视图、2D 面编辑、属性系统和状态管理之间形成一致的数据流。
5、参与前后端协作与工具链建设
基于 Node.js 支撑前端工程化、构建链路、数据接口联调和本地开发体验，提升复杂图形应用的可维护性、性能与协作效率。
职位要求
1、精通 TypeScript
具备扎实的类型设计能力和工程能力，能够在复杂状态、数据结构和组件协作中写出清晰、稳定、可维护的 TypeScript 代码。
2、熟悉 React
理解 React 生态下的组件设计、状态管理、性能优化和工程组织方式，能胜任复杂交互型应用而不只是内容展示型页面。
3、熟悉 Three.js
有实际 Web 3D 项目经验，理解场景、相机、材质、灯光、拾取、变换控制、性能优化等常见问题，能够把 Three.js 真正用于产品级交互而非简单 demo。
4、熟悉 Node.js
能够处理前端工程化、构建工具、脚本系统、开发服务和接口协作相关问题，对现代前端项目的运行机制有整体认知。
5、理解参数化结构，愿意和引擎深度协作
对图形编辑器、参数化建模、几何约束传播、结构联动这类问题有兴趣或经验。你不一定亲自实现几何内核，但需要理解前端如何承接这些能力，并把它们准确传达给用户。
6、对 Brep 有了解
了解 Brep、几何表达、拓扑关系或 CAD/建模相关基本概念者优先。我们希望你对“3D 形体背后是怎样被表示和操作的”有基本认知，而不只是停留在视觉层。
7、具备审美与产品感觉
你做过有质感的前端，理解什么是好的工具体验。对界面节奏、留白、层级、细节反馈、3D 画面表现和整体气质有判断力。会 3D、懂空间表达、有作品集者优先。

### 绘见科技-高级前端开发工程师（3D可视化方向）

一、岗位职责
1、负责3D可视化前端模块的设计、开发与优化，基于WebGL、Three.js、Cesium等技术构建高性能、高交互性的3D场景，支撑项⽬核⼼业务场景落地；
2、参与3D可视化技术⽅案的调研与制定，攻克大场景渲染、模型加载优化、光影效果适配、多端兼容等技术难点，提升产品体验与性能；
3、配合产品、设计及后端团队，完成需求对接、接口联调，保障3D功能与整体系统的协同运行，⾼效推进项目迭代；
4、负责3D可视化相关代码的规范编写、单元测试及文档整理，沉淀技术组件与开发经验，提升团队开发效率；
5、关注WebGL、Three.js、Cesium等前沿技术动态，引入合适的技术方案与优化思路，推动团队3D可视化技术能力升级；
6、处理3D场景在不同浏览器、不同设备上的兼容性问题，保障产品跨端稳定运行。
二、任职要求
1、本科及以上学历，计算机、软件工程等相关专业，5年及以上3D可视化方向专项开发经验，具备独立负责大型3D可视化项目的开发能力；
2、熟练掌握WebGL底层原理，能独立运用Three.js进行3D场景搭建、模型加载、动画制作、交互逻辑开发，具备复杂3D场景优化经验；
3、熟悉Cesium框架，能基于Cesium完成地理信息（GIS）3D可视化开发，包括地形加载、矢量数据渲染、空间查询、轨迹模拟等功能；
4、精通HTML5、CSS3、JavaScript/TypeScript，熟练掌握前端工程化（Webpack、Vite等）、模块化开发规范，具备良好的代码编写习惯；
5、了解3D模型格式（GLB、GLTF、FBX等）的特性及优化方法，能配合建模团队完成模型轻量化处理，提升场景加载速度与渲染性能。

### 深蓝信息-前端三维开发工程师

岗位职责：
1、负责基于WebGL、Three.js及Cesium框架的三维场景、地理信息可视化系统的核心模块开发、性能调优与体验提升。
2、处理海量三维模型（倾斜摄影、BIM）、矢量地理数据（GeoJSON、Shapefile）的加载、渲染与实时更新，确保在万级及以上数据量场景下的流畅交互。
3、利用原生WebGL及GLSL着⾊器编程，实现动态波浪、粒⼦系统、⾼级光照、后处理等定制化视觉效果。
4、设计和开发三维场景下的交互功能，如模型编辑、空间测量、剖切分析、实时数据动态联动等。
5、针对复杂三维技术难题进行攻关，设计并实施高性能、可扩展的技术解决⽅案。

任职要求：
1、本科及以上学历，计算机科学、软件工程、地理信息系统、数学或相关专业。
2、3年以上前端开发经验，其中包含2年以上专注三维或GIS可视化开发的经验。
3、具备良好的数学基础（线性代数、解析几何、矩阵变换），理解三维空间坐标、投影、光照等核心原理。
4、熟悉GIS相关概念（经纬度、投影坐标系、WGS84/GCJ02 坐标系转换），能处理地理空间数据（GeoJSON、Shapefile、KML）；
5、熟悉canvas以及WebGl，熟读WebGL编程指南，了解着色器语言glsl，能熟练运用各种内置函数进行编码，实现特殊视觉效果（如波浪、粒子、动态纹理）；
6、熟悉性能优化，对内存占用有一定了解，有一定的内存性能优化经历；
7、熟悉第三方开发框架three.js和Cesium。具备Cesium大型项目实战经验，曾处理过以下场景之一优先；
8、有海量3D Tiles数据（倾斜摄影、BIM） 的加载、调度、性能深度调优及自定义扩展经验优先。
9、在Cesium项目中，有万级以上Entity或Primitive实例的批量渲染与优化成功经验优先。
10、主导或核心参与过复杂三维交互功能（如模型构件级编辑、三维空间分析、实时传感器数据驱动场景更新）的开发。
11、有数字孪生、智慧城市、智慧园区、BIM+GIS融合应用项目开发经验。
12、对开源社区有贡献，或在WebGL/三维图形学方面有个人技术博客、作品集。

## 职位描述分析

我希望你能够结合上述的职位描述模块的内容和web 3D技术现状，分析出 web 3D 前端开发需要掌握的知识和技能。

不需要把上述涉及到的技术都列出来，这样太多了，可以给这些技术简单分类，简要说明技术的作用和能实现的效果，然后挑有代表性的进行学习。

如果是比较过时的技术，或者是目前3D行业使用很少的技术，对他们简单介绍就行，不需要学习。
